# Problem: Find the Difference

## 1. Problem Understanding

### Problem Summary

Given two strings `s` and `t`, where `t` is generated by shuffling `s` and adding one extra letter, find that extra letter.

### Input

- A string `s`
- A string `t`, where `t.length == s.length + 1`

### Output

- The extra character that was added to `t`.

### Constraints

- `0 <= s.length <= 1000`
- `t.length == s.length + 1`
- `s` and `t` consist of lowercase English letters.

### Example

Input:

```text
s = "abcd", t = "abcde"
```

Output:

```text
"e"
```

Manual walkthrough:

```text
s: a b c d
t: a b c d e

Every letter in s has a matching letter in t.
The letter 'e' in t has no match -> it's the extra letter.
```

---

## 2. Brute Force Approach

### Idea

Treat `t` as a mutable list of characters. For every character in `s`, remove one matching occurrence from that list. Whatever single character remains at the end is the extra letter.

### Pseudocode

```text
t_chars = list of characters in t

for char in s
    find the index of char in t_chars
    remove that entry from t_chars

return the one remaining character in t_chars
```

### Complexity Analysis

#### Time Complexity

```text
O(n^2)
```

Why?

- `n = len(t)`; for each of the `n - 1` characters in `s`, finding and removing a matching entry from `t_chars` costs `O(n)` (scan to find it, shift to close the gap).

#### Space Complexity

```text
O(n)
```

Why?

- `t_chars` is a full mutable copy of `t`.

### Why this isn't good enough

Every character removal re-scans and re-shifts the list, even though "how many of this character are available" is information that could be counted once up front. A frequency map built from `s` and consumed while scanning `t` turns each of those `O(n)` removals into an `O(1)` decrement.

---

## 3. Key Insight

### What makes this problem difficult?

`t` is a shuffled version of `s` plus one extra character, so the extra letter isn't necessarily at a fixed position — comparing characters index by index doesn't work directly.

### Key Observation

If we count how many times each character appears in `s`, then walk through `t` decrementing those counts, the one character that either isn't in the count map at all, or whose count has already hit zero, must be the extra letter.

Example:

```text
s = "abcd" -> freq = {a:1, b:1, c:1, d:1}

Scan t = "abcde":
a -> freq[a] 1->0
b -> freq[b] 1->0
c -> freq[c] 1->0
d -> freq[d] 1->0
e -> not in freq -> this is the extra letter
```

### Why does this observation help?

Instead of sorting both strings and comparing them character by character (`O(n log n)`), a hash map lets us count once and consume once, both in `O(n)` — the mismatch naturally surfaces as either a missing key or a spent count.

## 4. Mental Model

> What picture should I imagine in my head?

Picture a checklist built from `s` — one tally mark per letter. Walk through `t`, and for every letter, cross off one tally mark from the checklist. The first letter that has no tally marks left to cross off (or was never on the checklist) is the extra one.

```text
Checklist from s="abcd":  a:1  b:1  c:1  d:1

Walking t="abcde":
'a' -> cross off a's mark  (a:0)
'b' -> cross off b's mark  (b:0)
'c' -> cross off c's mark  (c:0)
'd' -> cross off d's mark  (d:0)
'e' -> no mark for 'e' on the checklist -> found it!
```

---

## 5. Decision Tree

```text
(Start)
   │
   ▼
Build freq from s (count each character)
   │
   ▼
For each char in t:
   │
   ▼
Is char in freq ?
   │
 ┌─┴─────────────────┐
 │                    │
No                    Yes
 │                    │
 ▼                    ▼
Return char       Is freq[char] == 0 ?
                       │
                     ┌─┴─────────┐
                     │            │
                    Yes           No
                     │            │
                     ▼            ▼
                Return char   freq[char] -= 1
                                   │
                                   ▼
                             Next char (or Done)
```

Explanation of each decision:

- A character in `t` that's completely absent from `freq` (never appeared in `s`) is immediately the extra letter.
- A character whose count has already dropped to `0` means every occurrence from `s` has already been matched — any further occurrence in `t` must be the extra one.
- Otherwise, decrement the count and continue — that occurrence is accounted for.

---

## 6. Plain English Algorithm

1. Build a frequency map `freq` counting every character in `s`.
2. Scan `t` left to right. For each `char`:
   - If `char` isn't in `freq`, it's the extra letter — return it.
   - If `freq[char]` is already `0`, every copy from `s` has been used up — return `char`.
   - Otherwise, decrement `freq[char]` and continue.

---

## 7. Pseudocode

```text
freq = empty map

for char in s
    freq[char] = freq.get(char, 0) + 1

for char in t
    if char not in freq
        return char
    if freq[char] == 0
        return char
    freq[char] -= 1
```

---

## 8. Python Solution

```python
class Solution:
    def findTheDifference(self, s: str, t: str) -> str:
        freq = {}
        for char in s:
            freq[char] = freq.get(char, 0) + 1

        for char in t:
            if char not in freq:
                return char

            if freq[char] == 0:
                return char

            freq[char] -= 1
```

## 9. Dry Run

Example:

```text
s = "abcd", t = "abcde"

freq built from s: {a:1, b:1, c:1, d:1}
```

| Step | char (from t) | freq[char] before | Action | Why? |
|------|---------------|---------------------|--------|------|
| 1 | 'a' | 1 | Not 0, decrement -> freq[a]=0 | Still had a copy left |
| 2 | 'b' | 1 | Not 0, decrement -> freq[b]=0 | Still had a copy left |
| 3 | 'c' | 1 | Not 0, decrement -> freq[c]=0 | Still had a copy left |
| 4 | 'd' | 1 | Not 0, decrement -> freq[d]=0 | Still had a copy left |
| 5 | 'e' | — | 'e' not in freq -> return 'e' | Never appeared in s |

Result: `"e"`

## 10. Complexity Analysis

### Time Complexity

```text
O(n)
```

Why?

- `n = len(t)`; one pass to build `freq` from `s` and one pass to scan `t`, each doing constant work per character.

### Space Complexity

```text
O(k)
```

Why?

- `k` is the number of distinct characters, bounded by the lowercase alphabet (at most 26).
