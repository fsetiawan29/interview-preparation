# LeetCode Trace Visualizer - Frontend Requirements

**Version:** 1.0  
**Framework:** React 19 + TypeScript + Vite  
**Purpose:** Interactive web application for visualizing LeetCode algorithm execution from JSON trace files.

---

# 1. Overview

Build a modern React application that visualizes algorithm execution generated by the Python Trace SDK.

The application should allow users to:

- Load a trace JSON file
- Step through algorithm execution
- Play/Pause execution like a video
- Inspect variables
- View algorithm decisions
- Visualize data structures
- Understand the algorithm intuitively

The application is **read-only**.

It does **NOT** execute Python code.

It only renders JSON produced by the Trace SDK.

---

# 2. Goals

The application should feel like a combination of:

- VS Code Debugger
- Python Tutor
- VisuAlgo
- Chrome DevTools Timeline

The UI should be polished enough that someone could use it during interview preparation or while creating educational content.

# 23. JSON Schema

The application should expect JSON similar to

```json
{
  "problem": "Longest Consecutive Sequence",
  "steps": [
    {
      "step": 1,
      "title": "Visit 100",
      "variables": {
        "current": 100,
        "length": 1,
        "longest": 1
      },
      "decision": {
        "condition": "100 - 1 not in set",
        "result": true,
        "action": "Start sequence"
      },
      "snapshot": {
        "type": "array",
        "values": [100,4,200,1,3,2],
        "highlight": [0],
        "pointers": {
          "current": 0
        }
      }
    }
  ]
}
```

The frontend must not assume only arrays exist.

It must support multiple snapshot types.
